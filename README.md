В соответствии с заданием разработано консольное приложение.
Для быстрой проверки в приложении имеется зарегистрированный пользователь login: Alice ; password: 11 и пользователь login: Alex; password: 11 .
У пользователей есть кошельки, в которых есть несколько операции доходов и расходов. У пользователя Alex проведены операции в соответствии с заданием "Пример работы". Так же в корневой директории проекта имеется файл "example.txt", в нем представлен пример листинга работы программы.
В приложении интуитивно понятная навигация, для перемещения по которому необходимо вводить цифры. Программа ведет сквозной подсчет бюджета, но имеется возможность выводить статистику по месяцам. Предупреждения о перерасходе месячных лимитов появляются в случае сортировки данных по месяцу.

Приложение поддерживает:

- проект использует Gradle;
- регистрацию и авторизацию пользователей;
- CLI-взаимодействие и цикл команд;
- добавление и учет оперций (доходов и расходов);
- кошелек привязанный к авторизованному пользователю, хранит баланс и все операции;
- бюджеты по категориям;
- вывод информации и статистики;
- уведомления в случае перерасходов и приближению к запланированному бюджету (более 80%);
- сохранение всех пользовательских данных в файл users.json;
- выгрузку в ходе загрузки (инициализации программы) всех пользоательских данных из файла users.json;
- проверку форматов, чисел, пустых значений, неизвестных категорий с понятными сообщениями;
- гибкий выбор категорий или периода, корректные уведомления при отсутствии данных;
- редактирование категорий с валидацией и контролем логики;
